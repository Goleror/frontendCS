# 🚀 Кроссплатформенный запуск проекта

Этот документ описывает все способы запуска проекта на разных операционных системах.

## Быстрый старт (все платформы) ⭐

### Рекомендуемый способ
```bash
npm run launch
```

Этот способ работает на **Windows**, **macOS**, **Linux** и **WSL** без изменений. Появится интерактивное меню выбора платформы.

## Альтернативные команды

### Linux / macOS
```bash
./run.sh
# или
npm run dev
```

### Windows
```cmd
npm run dev
```

## Как работает `npm run launch`?

1. **Интерактивное меню**: Выбирается платформа разработки (или авто-определение)
   - 🪟 Windows
   - 🐧 Linux
   - 🍎 macOS
   - 🔷 WSL
   - 🤖 Auto-detect

2. **Автоматические действия**:
   - ✅ Проверка наличия `node_modules`
   - 📦 Установка зависимостей если нужно (`npm install`)
   - 📁 Создание директории `logs/`
   - 🚀 Запуск сервера (`npm run dev`)
   - 🌐 Вывод информации о доступе

3. **Корректная работа на выбранной платформе**: Все переменные окружения и пути устанавливаются правильно для выбранной ОС

## Что делает каждый способ?

| Команда | Платформа | Описание |
|---|---|---|
| `npm run launch` | ✅ Все | Интерактивное меню выбора ОС |
| `./run.sh` | 🐧🍎 Linux/macOS | Bash скрипт с цветным выводом |
| `npm run dev` | ✅ Все | Прямой запуск dev сервера |

## Требования

- **Node.js** v16+
- **npm** или **yarn**

## Решение проблем

### На Linux/macOS скрипты не запускаются
```bash
chmod +x run.sh start.js
```

### На Windows cmd выводит ошибку типов
Убедитесь что используете `npm run launch` или просто `npm run dev`

### npm команды не работают
```bash
# Переустановите зависимости
rm -rf node_modules package-lock.json
npm install
```

## Дополнительные команды

```bash
# Запуск с выбором платформы (рекомендуется)
npm run launch

# Прямой запуск dev сервера
npm run dev

# Проверить типы TypeScript
npm run check

# Собрать продакшн версию
npm run build

# Запустить продакшн версию
npm start

# Синхронизировать БД (Drizzle)
npm run db:push
```

## Переменные окружения

По умолчанию используется:
- `NODE_ENV=development`
- `PORT=5000` (через vite.config.ts)

Для изменения порта отредактируйте `vite.config.ts`:

```typescript
server: {
    host: "0.0.0.0",
    port: 5000,  // 👈 Измените здесь
    middlewareMode: false,
}
```
