# Реализация системы ролей и управления классами

## 📋 Описание
Реализована система регистрации и управления пользователями с разделением на роли: **ученик** и **учитель**. Добавлена функциональность управления классами через 5-значные коды доступа и автоматическая инициализация админского аккаунта.

---

## 🔧 Внесённые изменения

### 1. **Database Schema** (`shared/schema.ts`)

#### Новые поля в таблице `users`:
- `role` (TEXT): "student" или "teacher" - роль пользователя
- `class_code` (TEXT): 5-значный код для студента (входящий) или кода класса учителя

#### Новые таблицы:
- **`classes`** - информация о классах, созданных учителями
  - `id` (INTEGER PRIMARY KEY)
  - `teacher_id` (INTEGER) - ссылка на учителя
  - `class_code` (TEXT UNIQUE) - 5-значный код класса
  - `class_name` (TEXT) - имя класса
  - `created_at` (TEXT) - дата создания

- **`class_students`** - связь студентов с классами
  - `id` (INTEGER PRIMARY KEY)
  - `student_id` (INTEGER) - ссылка на студента
  - `class_id` (INTEGER) - ссылка на класс
  - `joined_at` (TEXT) - дата присоединения

#### Новые типы в Zod:
- `insertClassSchema`, `selectClassSchema`
- `insertClassStudentSchema`, `selectClassStudentSchema`

## 📊 Логика регистрации

### Сценарий 1: Регистрация ученика
```
1. Пользователь выбирает роль "Ученик"
2. Вводит код класса (5 символов) - XXXXX
3. Система проверяет: существует ли класс с кодом XXXXX
4. Если ДА:
   - Создаётся пользователь со role="student"
   - Ученик добавляется в таблицу class_students
5. Если НЕТ: 
   - Ошибка "Class with this code not found"
```

### Сценарий 2: Регистрация учителя
```
1. Пользователь выбирает роль "Учитель"
2. Система генерирует 5-значный код: например 75432
3. Создаётся пользователь со role="teacher", class_code="75432"
4. В таблице classes создаётся запись для этого класса
5. Код отображается на экране для передачи ученикам
```

## 🔒 Безопасность

- ✅ Пароли хешируются с bcryptjs (10 раундов)
- ✅ Валидация всех входных данных
- ✅ Проверка существования кода класса перед добавлением студента
- ✅ Коды классов генерируются случайно (5 цифр: 10000-99999)
- ✅ Admin пароль мощный (содержит прописные буквы, цифры)

| Метод | Endpoint | Параметры | Возвращает |

| POST | `/api/auth/register` | `username`, `password`, `role`, `classCode?` | Пользователь с кодом |

| GET | `/api/auth/user` | - | Информация о пользователе и прогресс |

## 📝 Примечания

- При первом запуске сервера автоматически создаётся админский аккаунт
- Коды классов уникальны (UNIQUE constraint в БД)
- Студент может быть в одном классе (хранится в class_code)
- Каждый класс привязан к одному учителю
- Все данные сохраняются в SQLite БД (newarch.db)
